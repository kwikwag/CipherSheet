How to Use CipherSheet

1. Generate a Key

Click "Generate new key" in the add-on sidebar. A .vaultkey file containing your 256-bit encryption key will instantly download to your machine. Do not lose this file.

2. Load & Protect

Select any cell, type your sensitive data into the CipherSheet text field, and click Protect. The cell is instantly replaced with ciphertext directly in the spreadsheet.

3. Decrypt on Demand

Collaborators who load the correct .vaultkey file into their sidebar will instantly see the decrypted contents of selected cells within the sidebar view.

4. Revealing & Clearing

No longer a secret? You can "Reveal" the plaintext back into the sheet, or "Clear" the cell to remove the ciphertext without decrypting it.

Honest Security Guarantee

CipherSheet uses a zero-knowledge architecture for key handling. This means:

  • Client-side encryption: All encryption and decryption occurs locally in your browser using the Web Crypto API.
  • Keys stay with you: Your .vaultkey is never transmitted to us or to Google servers. The add-on only sends the resulting, unintelligible ciphertext to the cloud.
  • Data loss warning: Because we have no access to your keys, we have no way to recover encrypted data if you lose your key file. Keep it in a secure password manager or physical drive.

Why We Need Permissions

When you install CipherSheet, Google will ask you to authorize specific permissions to your account. We only ask for what is strictly necessary:

  • Read & Write to Spreadsheets (https://www.googleapis.com/auth/spreadsheets.currentonly): We strictly ask for access to the current spreadsheet only. We require this to read the selected cell's value and inject the AES-256 ciphertext back into the sheet when you click "Protect" or "Unprotect".
  • Display internal add-on UI: required to create the custom Add-on Sidebar and Menu you see.
