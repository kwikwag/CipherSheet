CipherSheet Privacy Policy

Last updated: March 20, 2026

This Privacy Policy applies to the CipherSheet Google Sheets add-on and the CipherSheet website. It explains how CipherSheet accesses, uses, stores, and shares Google user data.

Who We Are

CipherSheet is an independent open-source project. Project homepage: https://kwikwag.github.io/CipherSheet/.

Plain-Language Terms

  • Google user data: data obtained through Google APIs used by CipherSheet.
  • Process: access, read, write, or otherwise handle data to provide app features.
  • Share or disclose: make data available to another party outside normal operation of Google services used by the app.
  • Retain: keep data until it is changed, deleted, or automatically expires.

Google User Data We Access

CipherSheet requests the minimum Google OAuth scopes needed for user-facing features.

  • auth/spreadsheets.currentonly: Read and write selected cell values in the current spreadsheet, and manage CipherSheet notes and cell editing protection.
  • auth/script.container.ui: Show CipherSheet sidebar and dialogs in Google Sheets.
  • auth/userinfo.email: When audit logging is enabled, record the acting user email in the audit log sheet.

How We Use Google User Data

CipherSheet only uses Google user data to perform the functionality provided by the add-on.

  • Encrypt plaintext entered by the user and write ciphertext to selected cells.
  • Decrypt ciphertext locally in the browser when a valid key is loaded.
  • If you choose Reveal, send plaintext to Apps Script only to write it back to the selected cell.
  • Optionally maintain an in-spreadsheet audit log when enabled by the sheet owner.
  • Store add-on settings in Apps Script document properties.

What We Do Not Do With Google User Data

  • We do not transfer Google user data outside Google Sheets.
  • We do not sell Google user data.
  • We do not use Google user data for advertising (including targeted or interest-based advertising).
  • We do not transfer Google user data to data brokers, information resellers, or for credit or lending purposes.
  • We do not use Google user data to train generalized AI/ML models.

Sharing and Disclosure

CipherSheet does not run its own backend service. CipherSheet does not store your spreadsheet content or keys.

  • We do not disclose Google user data to third parties for unrelated purposes.
  • Data needed for add-on functionality is processed through Google Sheets and Google Apps Script.
  • Spreadsheet collaborators can view spreadsheet content according to your Google Sheets sharing settings.
  • Unless explicitly approved by the user, plaintext data is kept on the client (user browser) and not communicated to Google Sheets.

Storage, Retention, and Deletion

  • CipherSheet does not store spreadsheet content or Google user data on CipherSheet-owned servers.
  • Encryption keys are generated and stored locally by the user's browser. They are not communicated with Google Sheets nor the CipherSheet add-on.
  • Document settings and optional audit logs remain until changed or deleted by users with spreadsheet access.
  • Temporary cache entries used for confirmation flow are short-lived and expire automatically.

You can delete data by clearing relevant cells, deleting the audit log sheet, removing document properties, or deleting the spreadsheet file. Uninstalling the add-on does not automatically delete existing spreadsheet data.

Security Protections

  • Client-side encryption/decryption uses the browser Web Crypto API and AES-256-GCM.
  • The key file is not uploaded by CipherSheet and is not recoverable by CipherSheet if lost.
  • OAuth scopes are limited to what is required for user-facing functionality.
